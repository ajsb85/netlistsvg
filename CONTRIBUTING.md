# Contributing to Netlist2SVG

Thank you for your interest in contributing to Netlist2SVG! This document provides instructions for developers looking to contribute code, fix bugs, or manage releases.

## Development Setup

1.  **Clone the repository:**
    ```sh
    git clone https://github.com/ajsb85/netlist2svg
    cd netlist2svg
    ```
2.  **Install dependencies:**
    ```sh
    npm install
    ```
3.  **Build the project:**
    ```sh
    npm run build
    ```
4.  **Run tests:**
    ```sh
    npm test
    ```

## NPM Publishing (Maintainers Only)

Netlist2SVG uses **NPM Trusted Publishing** via OpenID Connect (OIDC). This allows GitHub Actions to publish directly to NPM without long-lived tokens, improving security and providing cryptographic provenance.

### Setting up Trusted Publishing

If you are a maintainer and need to re-configure or set up the trust relationship between GitHub and NPM, follow these steps:

1.  **Login to NPM:**
    Ensure you are logged in to the account that owns the package.
    ```sh
    npm login
    ```

2.  **Establish the Trust Relationship:**
    Run the following command from your local machine. This will open a browser window for interactive Two-Factor Authentication (2FA).
    ```sh
    npm trust github --file publish.yml --repo ajsb85/netlist2svg --yes
    ```
    *   `--file publish.yml`: Refers to the workflow file in `.github/workflows/`.
    *   `--repo ajsb85/netlist2svg`: The specific repository authorized to publish.

3.  **Verify Configuration:**
    Once authenticated, NPM will trust the OIDC tokens generated by the GitHub Actions `publish.yml` workflow. You can verify the existing trust relationships with:
    ```sh
    npm trust list netlist2svg
    ```

### Automated Release Workflow

The project is configured to publish automatically whenever a new version tag is pushed:

1.  Update the version in `package.json`.
2.  Commit the change.
3.  Create and push a tag:
    ```sh
    git tag v1.x.x
    git push origin v1.x.x
    ```
4.  The `Publish Package` workflow will trigger, run tests, and publish to NPM with `--provenance`.

## Code Style

- Use **Single Quotes** for strings.
- Ensure all TypeScript types are correctly defined.
- Run `npm run lint` before committing.
